# Data Directory

Raw and processed data are **not committed** to the repository (too large and
subject to upstream license terms). You have two options for populating this
directory.

---

## Option A — Download the pre-built data bundle (recommended)

A pre-built archive containing the raw OpenAlex papers, the news-article
corpus, and the train/val/test splits is available on Baidu Netdisk:

- **Link**: <https://pan.baidu.com/s/16KhHWR2VLoiILtkHzN79lw?pwd=b6xw>
- **Access code**: `b6xw`
- **Share name**: *Science-Newsworthy*

Extract the archive into the repository root so the final layout is:

```
data/
├── raw/
│   ├── medicine/
│   │   ├── media_data.json              # positive + negative for media task
│   │   ├── policy_data.json             # positive + negative for policy task
│   │   └── openalex_papers.json         # (optional) full OpenAlex dump
│   ├── environmental_science/           # same three files
│   ├── social_sciences/                 # same three files
│   ├── ced/                             # Crossref Event Data monthly dumps
│   │   ├── newsfeed_YYYY_MM.json
│   │   └── news_dois.json
│   └── news_text/
│       └── news_articles.json           # ~88 K fetched news-article texts
└── processed/                           # 70 / 15 / 15 train/val/test splits
    ├── medicine/
    │   ├── media/{train,val,test}.json
    │   └── policy/{train,val,test}.json
    ├── environmental_science/
    └── social_sciences/
```

With this in place you can skip `step1` (data collection) and `step2`
(cleaning/splitting) entirely. Proceed to `step4` (SFT formatting) — it will
create `data/sft/` from `data/processed/`.

---

## Option B — Re-collect from the source APIs

If you prefer to rebuild everything from scratch:

1. Set `OPENALEX_API_KEY` in the environment (or edit `config.py`).
2. Run:
   ```bash
   bash run_pipeline.sh step1      # OpenAlex + Crossref Event Data
   bash run_pipeline.sh step2      # cleaning + 70/15/15 split
   bash run_pipeline.sh step3      # CoT generation (requires Anthropic API key)
   bash run_pipeline.sh step4      # SFT formatting
   ```

Expected runtime: several hours for `step1` (rate-limited APIs).

---

## What is **not** in the bundle

- `data/sft/` — regenerated by `scripts/step4_format_sft.py`
- LoRA adapters, predictions, analysis caches — produced by the training /
  evaluation / analysis stages (see the top-level `README.md`)
