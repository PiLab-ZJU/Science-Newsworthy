# Predicting and Interpreting Media Coverage of Scientific Papers via LLMs

This repository contains the code for an empirical study on predicting the
**societal impact of scientific papers** — specifically whether a paper will
receive mainstream news coverage or be cited in public-policy documents —
directly from its title and abstract, using supervised fine-tuning (SFT) of
open-weight large language models.

Beyond prediction, we analyse *what* the model has learned: we compare the
"news-value signals" extracted from model explanations with the signals found
in real news coverage, following the news-value framework of Harcup & O'Neill
(2017).

> **Target venue**: Scientometrics — *LLM4SCIM* special issue
> (LLMs for altmetrics / LLM-empowered prediction models)

## 1. Overview

**Tasks.**

- **Media-coverage prediction** — binary: will this paper be covered by
  mainstream news outlets? (label source: Crossref Event Data)
- **Policy-citation prediction** — binary: will this paper be cited in a public
  policy document? (label source: Overton via Altmetric)

**Models.**

- Base: `meta-llama/Llama-3.1-8B-Instruct` (primary), `Qwen/Qwen2.5-7B-Instruct`
  (alternate)
- Fine-tuned with LoRA via [LLaMA-Factory](https://github.com/hiyouga/LLaMA-Factory)
  (rank 16/32, alpha 32, dropout 0.05)

**Framework.** A two-step **Predict-then-Interpret** pipeline:

1. *Predict* — SFT LLM outputs Yes/No + a Chain-of-Thought rationale.
2. *Interpret* — rationales are clustered, aligned to news-value categories,
   and compared against a corpus of ~88 K news articles that actually covered
   scientific papers.

**Data.**

- ~160 K labelled papers from OpenAlex (Medicine as primary field;
  Environmental Science and Social Sciences for cross-domain evaluation)
- ~88 K news-article texts for the model-vs-news vocabulary/signal comparison
- Raw data is **not committed** (see `data/README.md`)

**Headline result.**

| Task  | Model                        | Accuracy | F1   |
|-------|------------------------------|----------|------|
| Media | Llama-3.1-8B + LoRA (CoT)    | 0.80     | 0.79 |
| Media | SciBERT / traditional ML     | —        | see `baselines/` |
| Media | Llama-3.1-8B zero-shot       | —        | see `baselines/llm_zeroshot.py` |

(Full tables are regenerated by `analysis/` scripts once predictions exist.)

## 3. Installation

Requires Python ≥ 3.10 and (for SFT training) a CUDA-capable GPU.

```bash
git clone <this-repo>
cd <this-repo>

python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

For SFT training, also install [LLaMA-Factory](https://github.com/hiyouga/LLaMA-Factory)
separately — the YAMLs in `configs/` are consumed by its `llamafactory-cli`.

## 5. End-to-end pipeline

All stages can be run individually or chained via `run_pipeline.sh`:

```bash
bash run_pipeline.sh step0          # Data exploration
bash run_pipeline.sh step1          # OpenAlex + Crossref collection
bash run_pipeline.sh step2          # Cleaning + train/val/test split (70/15/15)
bash run_pipeline.sh step3          # CoT rationale generation
bash run_pipeline.sh step4          # SFT formatting
bash run_pipeline.sh baselines      # Traditional ML + SciBERT baselines
bash run_pipeline.sh train          # (manual) LLaMA-Factory SFT
bash run_pipeline.sh eval           # Inference + metrics
bash run_pipeline.sh analysis       # Keyword / signal / ablation analysis
bash run_pipeline.sh cross_domain   # Transfer to Environmental / Social Sciences
```

### Training (LLaMA-Factory)

```bash
# copy SFT files into LLaMA-Factory's data directory
cp data/sft/medicine/media_train.json  LLaMA-Factory/data/
cp data/sft/medicine/policy_train.json LLaMA-Factory/data/
cp data/sft/medicine/joint_train.json  LLaMA-Factory/data/

# merge configs/dataset_info.json into LLaMA-Factory/data/dataset_info.json

llamafactory-cli train configs/media_lora_sft.yaml
llamafactory-cli train configs/policy_lora_sft.yaml
llamafactory-cli train configs/joint_lora_sft.yaml
```

Trained LoRA adapters are written to `outputs/<task>_lora_sft/`.

### Inference + metrics

```bash
python evaluation/inference.py --task media  --adapter_path outputs/media_lora_sft
python evaluation/inference.py --task policy --adapter_path outputs/policy_lora_sft
python evaluation/metrics.py --predictions outputs/predictions/medicine/media_test_predictions.json
```

### Baselines

```bash
python baselines/traditional_ml.py --task media
python baselines/scibert_baseline.py --task media
python baselines/specter2_baseline.py --task media
python baselines/llm_zeroshot.py --task media --model gpt4o --mode zero_shot --max_test 500
```

### Analysis

```bash
python analysis/keyword_analysis.py --task compare
python analysis/signal_taxonomy.py \
    --media_predictions  outputs/predictions/medicine/media_test_predictions.json \
    --policy_predictions outputs/predictions/medicine/policy_test_predictions.json
python analysis/paper_figures.py   # regenerates figures under outputs/figures/
```

## 6. What is *not* in this repository

To keep the repo small and respect upstream licensing, the following are **not**
committed. The directories are kept as empty placeholders with local READMEs
that explain how to regenerate their contents.

- `data/raw/`, `data/processed/`, `data/sft/`, `data/news/` — raw and cleaned
  paper/news data (download from Baidu Netdisk, see the *Data download*
  section at the top)
- `outputs/*_lora_sft/` — LoRA adapter weights (run SFT to produce)
- `outputs/predictions/` — test-set predictions
- `outputs/figures/`, `outputs/tables/` — regenerable
- `analysis/*.npy`, `analysis/*.json` — cached embeddings, signal tables, topic
  models (see `analysis/cached_artifacts/README.md`)
- `logs/` — training / evaluation logs
- Base LLM weights — pulled from Hugging Face on first use
